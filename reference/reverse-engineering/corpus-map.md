## File Roles

- `metadata.txt`: binary fingerprint, program metadata, and export counts.
- `functions.csv`: function inventory sorted by body size for prioritizing large subsystems.
- `imports.txt`: imported engine/runtime edges.
- `exports.txt`: DLL export table entry points.
- `analysis_symbols.txt`: analyst-defined or imported non-default symbol names.
- `decompile_exports.c`: direct decompilation of exported entry points, especially `GetGameAPI`.
- `decompile_top_functions.c`: broad subsystem hint set for large function recovery.
- `selected_functions_summary.csv`: targeted address-to-symbol map for export callbacks and other high-value functions, including a `renamed` column that records when an existing exact-address function was promoted to a stable retail name in the temporary analysis project.
- `decompile_selected_functions.c`: direct decompilation for the targeted address set above.
